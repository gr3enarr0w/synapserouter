# Changelog

## v1.05 — Performance + Model Optimization + Deep Search (2026-04-02)

### Speed Optimizations
- K-LLM parallel verification: K independent reviewers with shuffled diffs, finding merge by agreement (#290)
- Context compression: structured summaries, observation masking, 70% context-fill trigger (#372)
- Diff-based review: send git diffs instead of full files, 60-80% token savings (#295)
- Plan caching: keyword-based SQLite cache, ~50% cost savings on similar tasks (#371)
- PASTE speculative execution: predict and pre-execute read-only tools while LLM thinks (#297)

### Search
- Search fusion: RRF merge across all configured backends in parallel (#379)
- 19 search backends: Brave, Tavily, Serper, Exa, SearXNG, DuckDuckGo, You.com, SerpAPI, SearchAPI.io, Jina, Kagi, Linkup, Semantic Scholar, OpenAlex, GitHub Search, Sourcegraph, NewsAPI, Newsdata.io, TheNewsAPI (#380)
- Dynamic backend selection by query type (code/academic/news/general)

### Deep Research Pipeline
- `/research [quick|standard|deep] <query>` REPL command (#365)
- Multi-round search with saturation detection and budget enforcement
- Free-first backend ordering, citation-aware synthesis
- 3 depth tiers: quick ($0), standard (<$0.05), deep (<$0.50)

### Model & Config
- `synroute recommend` command: detects providers, suggests tier configs (#369)
- YAML tier config: `~/.synroute/config.yaml` or `.synroute.yaml` (#370)
- `synroute config show/generate` commands
- Bundled model catalog with benchmark scores for ~30 models

### Benchmark Tooling
- Latency percentiles (p50/p95/p99) in eval results (#373)
- Cost tracking per exercise and per run
- Cost efficiency ratio (pass_rate / cost)
- Config-vs-config A/B testing support

### Other
- Renamed Go module from mcp-ecosystem/synapse-router to gr3enarr0w/synapserouter
- LLM provider comparison doc (generated by synroute's own research pipeline)
